Machine learning is a field of artificial intelligence that focuses on building systems that learn from data. The three main types of machine learning are supervised learning, unsupervised learning, and reinforcement learning. Supervised learning involves training on labeled data, while unsupervised learning finds patterns in unlabeled data. Reinforcement learning uses rewards and punishments to train agents.

Overfitting is a common problem where a model learns the training data too well, including its noise and outliers, which prevents it from generalizing to new, unseen data. This is often linked to the bias-variance tradeoff: high bias leads to underfitting (the model is too simple), while high variance leads to overfitting (the model is too complex). To prevent overfitting, techniques like regularization (L1 and L2) are used, which add penalty terms to the loss function to discourage complexity. Cross-validation is another essential technique, splitting data into multiple folds to get a more reliable estimate of model performance.

In neural networks, backpropagation is the core algorithm for training, using the chain rule to compute gradients of the loss function with respect to weights. Common activation functions include ReLU, sigmoid, and tanh. Convolutional Neural Networks (CNNs) are primarily used for grid-like data such as images, using pooling layers to reduce spatial dimensions and complexity. Recurrent networks like LSTMs address the vanishing gradient problem using gating mechanisms.

Natural Language Processing (NLP) has been revolutionized by the transformer architecture, introduced in the 'Attention Is All You Need' paper in 2017. Transformers use self-attention to weigh the importance of different words in a sequence. Word embeddings represent words as dense vectors in a continuous space. Transfer learning involves pre-training models like BERT or GPT on large datasets and then fine-tuning them for specific tasks. BERT uses bidirectional training, while GPT is autoregressive (predicts the next token). GPT-4 features a massive context length of up to 128K tokens. Techniques like instruction tuning and RLHF (Reinforcement Learning from Human Feedback) are used to align LLMs with human intent. Chain-of-thought prompting helps models reason through complex problems step-by-step.

Retrieval-Augmented Generation (RAG) is a technique that combines generative LLMs with external knowledge retrieval. A RAG pipeline typically consists of a retriever and a generator. Dense retrieval uses embeddings to find relevant passages, while chunking strategy (fixed-size, semantic, or hierarchical) determines how documents are split. Advanced RAG techniques include re-ranking, query expansion, and HyDE. Vector databases like FAISS, using algorithms like HNSW, are used to store and query these high-dimensional embeddings. Hybrid search combines vector similarity with keyword-based search.

LangChain is a popular framework for building LLM applications, featuring LangChain Expression Language (LCEL) for composing chains. LangGraph extends this for stateful, multi-actor graphs. LangSmith is a companion platform for tracing, debugging, and monitoring these applications. It captures inputs, outputs, and latency, and includes a Prompt Hub for versioning templates.

RAGAS (Retrieval Augmented Generation Assessment) is a framework for automated evaluation of RAG pipelines. It computes metrics like faithfulness (checking if claims are in the context), answer relevancy, context recall, and context precision.

AI safety is critical, with tools like Guardrails AI providing validators to detect PII (Personally Identifiable Information) like emails or phone numbers, and ensuring structured outputs like JSON are valid. Constitutional AI involves giving models principles to follow for safer behavior. Common concerns include hallucinations, toxicity, and bias.
